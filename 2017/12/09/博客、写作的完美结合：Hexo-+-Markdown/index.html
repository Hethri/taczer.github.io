博客、写作的完美结合：Hexo + Markdown

发表于 2017-12-09

前言

起初因为被Markdown格式的写作风格吸引：简练，无需刻意排版，支持数学公式渲染和代码高亮，支持导出为PDF等格式方便跨设备使用，它既不像Word那样需要刻意的排版以至于打扰到写作，也不至于像Latex那样需要繁琐的配置，让你能直接上手写作，而不是花费时间在其他地方。

话虽如此，当正式写的时候你就会发现有很多问题了。首先是找到一款颜值高且功能完备（满足我的需求）的Markdown编辑器。

Markdown编辑器推荐

说明：以下Markdown编辑器包含通过组合方式实现功能的且只讨论Windows平台

Typora

优点：

首先，这篇文字就是在Typora上写就的。Typora拥有所有我所必须的功能：数学公式支持，代码高亮，导出多种格式，方便在手机上，iPad上观看。除此以外，Typora的颜值也相当了得，在使用过大概十多款编辑器后，还是它最受我的喜爱。

缺点：

不支持云同步。目前的解决方案是云坚果云进行同步。

LitteStory（小书匠）

优点：

小书匠应该是我使用频率最高的一款Markdown编辑器了，它功能之丰富，目前在这个领域应该还没有什么敌手。基本功能：数学公式、代码高亮、多格式导出、云同步、个性化导出、编辑器主题丰富

缺点：

界面设计不够好，有选项之间会遮挡等问题的存在。

有道云笔记

优点：

格式支持齐全，支持多设备云同步。支持网页剪藏功能，微信、微博页面保存。多终端支持

缺点：

有广告

为知笔记

优点：

格式支持齐全，有插件扩展，支持多种编辑器，可定制。多终端支持

缺点：

需要付费

Yu Writer

优点：

格式支持齐全，

缺点：

没有云听不

Moeditor

优点：

格式支持齐全

缺点：

不支持多设备云同步。

###VsCode+markdown preview enhanced

优点：

格式支持齐全，无缝衔接到代码说明，通过插件支持导出到PDF

缺点：

需要配置，打开速度不如sublime，没有同步

Sublime+markdown preview

优点：

格式支持齐全，打开速度快，写作体验好

缺点：

需要配置，无云同步

Atom+markdown preview enhanced

优点：

格式支持齐全，界面现代化

缺点：

无云同步，打开速度慢

Visual Studio 2017+Markdown Support

优点：

支持数学公式，支持在VS下写Markdown文档

缺点：

在VS下就不要求同步了吧······

蚂蚁笔记

优点：

格式支持齐全，支持多设备云同步。自带博客支持，将写好的markdown文档发布成为blog

缺点：

收费，界面设计不够漂亮

作业部落

优点：

格式支持齐全

缺点：

收费才能导出PDF

OneNote

优点：

格式支持齐全，通过插件可以解决所有所需

缺点：

写公式的时候体验不好，复制latex公式时不能转化，只能手打
